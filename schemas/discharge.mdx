## Overview

The DISCHARGE schema captures information about the end of a treatment episode. A discharge record closes an open admission and allows for outcome tracking. Each admission can have at most one discharge record.

### Invalid Example

```csv
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,BHASO_CASE_IDENTIFIER,EFFECTIVE_DATE,DISCHARGE_DATE,DISCHARGE_REASON,DISCHARGE_STATUS,LIVING_SITUATION,EMPLOYMENT_STATUS,TRANSFER_FACILITY
DIS,A,CLT-000001,CSE-999999,2024-06-15,2024-06-15,01,01,01,02,
DIS,A,CLT-000001,CSE-000001,2024-06-15,2024-01-01,01,01,01,02,
DIS,A,CLT-000001,CSE-000002,2024-06-15,2024-06-15,04,04,01,02,
```

**Errors:**
- Row 1: `REF-002` - Admission not found: CSE-999999 (must submit ADMISSION record first)
- Row 2: `DIS-005` - DISCHARGE_DATE (2024-01-01) cannot be before ADMISSION_DATE (2024-01-15)
- Row 3: `XFD-001` - TRANSFER_FACILITY required when DISCHARGE_REASON = 04

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `DIS-001` | BHASO_CASE_IDENTIFIER is required | Provide case identifier |
| `DIS-002` | DISCHARGE_DATE is required | Provide discharge date |
| `DIS-003` | DISCHARGE_REASON is required | Provide valid discharge reason code |
| `DIS-004` | Admission not found | Submit ADMISSION record first |
| `DIS-005` | DISCHARGE_DATE cannot be before ADMISSION_DATE | Check date values |
| `DIS-006` | Discharge already exists for this admission | Use Update (U) to modify |
| `DIS-007` | DISCHARGE_DATE cannot be in future | Use date on or before today |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Transfer | `DISCHARGE_REASON`, `TRANSFER_FACILITY` | If `DISCHARGE_REASON` = `04`, `TRANSFER_FACILITY` required |
| Death | `DISCHARGE_REASON`, `DEATH_DATE` | If `DISCHARGE_REASON` = `06`, `DEATH_DATE` required |
| Incarceration | `DISCHARGE_REASON`, `INCARCERATION_FACILITY` | If `DISCHARGE_REASON` = `05`, `INCARCERATION_FACILITY` required |
| Follow-up | `FOLLOWUP_PLANNED`, `FOLLOWUP_DATE` | If `FOLLOWUP_PLANNED` = `1`, `FOLLOWUP_DATE` recommended |
| Date Sequence | `DISCHARGE_DATE`, `FOLLOWUP_DATE` | `FOLLOWUP_DATE` must be after `DISCHARGE_DATE` |

## 6. Update Semantics

### Primary Key

`BHASO_CASE_IDENTIFIER` (one discharge per admission)

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates discharge record; fails if discharge exists for case |
| Update | `U` | Updates existing discharge record |
| Replace | `R` | Overwrites discharge record |

### Idempotency

- Only one discharge record per `BHASO_CASE_IDENTIFIER`
- Once discharged, case cannot receive new clinical records
- Re-admission requires new `BHASO_CASE_IDENTIFIER`
