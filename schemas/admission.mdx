## Overview

The ADMISSION schema captures information about the start of a treatment episode. Each admission creates a unique `BHASO_CASE_IDENTIFIER` that links all related clinical records (diagnoses, substance use, discharge, etc.).

### Invalid Example

```csv
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,BHASO_CASE_IDENTIFIER,EFFECTIVE_DATE,ADMISSION_DATE,BHASO_IDENTIFIER,SERVICE_IDENTIFIER,REFERRAL_SOURCE,LEGAL_STATUS,LIVING_SITUATION,EMPLOYMENT_STATUS,PRIOR_TREATMENT,PRIOR_EPISODE_DATE,COURT_CASE_NUMBER
ADM,A,CLT-999999,CSE-000006,2024-01-15,2024-01-15,BHA-001,001,01,01,01,01,0,,
ADM,A,CLT-000001,CSE-000001,2024-01-15,2024-01-15,BHA-001,001,01,03,01,01,1,,
```

**Errors:**
- Row 1: `REF-001` - Client not found: CLT-999999 (must submit CLIENT record first)
- Row 2: `ROW-003` - Duplicate primary key (BHASO_CASE_IDENTIFIER + EFFECTIVE_DATE already exists)
- Row 2: `XFD-001` - COURT_CASE_NUMBER required when LEGAL_STATUS = 03
- Row 2: `XFD-001` - PRIOR_EPISODE_DATE required when PRIOR_TREATMENT = 1

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `ADM-001` | BHASO_CASE_IDENTIFIER is required | Provide unique case identifier |
| `ADM-002` | ADMISSION_DATE must be valid date | Use YYYY-MM-DD format |
| `ADM-003` | BHASO_IDENTIFIER is required | Provide valid BHASO provider ID |
| `ADM-004` | SERVICE_IDENTIFIER is required | Provide valid service type code |
| `ADM-005` | Client does not exist | Submit CLIENT record first |
| `ADM-006` | ADMISSION_DATE cannot be before DOB | Check admission and birth dates |
| `ADM-007` | ADMISSION_DATE cannot be future | Use date on or before today |
| `ADM-008` | Duplicate BHASO_CASE_IDENTIFIER | Case ID must be unique per episode |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Prior Treatment | `PRIOR_TREATMENT`, `PRIOR_EPISODE_DATE` | If `PRIOR_TREATMENT` = `1`, `PRIOR_EPISODE_DATE` required |
| Legal Status | `LEGAL_STATUS`, `COURT_CASE_NUMBER` | If `LEGAL_STATUS` in (`02`, `03`, `04`), `COURT_CASE_NUMBER` required |
| Dependents | `HAS_DEPENDENTS`, `NUM_DEPENDENTS` | If `HAS_DEPENDENTS` = `1`, `NUM_DEPENDENTS` required (1-99) |
| Pregnancy | `SEX` (from CLIENT), age, `PREGNANT` | If female and age 12-55, `PREGNANT` required |
| Date Order | `ADMISSION_DATE`, `PRIOR_EPISODE_DATE` | `PRIOR_EPISODE_DATE` must be before `ADMISSION_DATE` |

## 6. Update Semantics

### Primary Key

`BHASO_CASE_IDENTIFIER` + `EFFECTIVE_DATE`

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates new admission record |
| Update | `U` | Creates new version with updated effective date |
| Replace | `R` | Overwrites most recent admission record |

### Idempotency

- `BHASO_CASE_IDENTIFIER` links all records within a treatment episode
- Once assigned, `BHASO_CASE_IDENTIFIER` cannot be changed
- Updates create new versions; original records retained
