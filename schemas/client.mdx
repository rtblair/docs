## Overview

The CLIENT schema captures core demographic information for individuals receiving behavioral health services. This is the root record type - all other records reference a client through the `BHASO_CLIENT_IDENTIFIER`.

## 1. Schema Reference

### Required Fields

| BHA Column Name | Field Name | Format | Valid Values | Constraints | Example |

| `EFFECTIVE_DATE` | Effective Date | `date` | YYYY-MM-DD | Cannot be future | `2024-01-15` |
| `DOB` | Date of Birth | `date` | YYYY-MM-DD | Must be ≤ today, age ≤ 120 | `1990-05-15` |
| `LAST_NAME` | Last Name | `x(50)` | Text | Required | `Smith` |
| `FIRST_NAME` | First Name | `x(50)` | Text | Required | `Jane` |

| `RACE` | Race | `x(50)` | See [Race/Ethnicity](/reference/appendix-a/race-ethnicity) | Multi-value (`; ` delimited) | `05; 06` |

## 2. Examples

### Minimal Valid Example

```csv
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,EFFECTIVE_DATE,DOB,LAST_NAME,FIRST_NAME,SEX,RACE,ETHNICITY,COUNTY_CODE
CLI,A,CLT-000001,2024-01-15,1990-05-15,Smith,Jane,F,05,2,031
```

```csv
# International characters in name
```

### Invalid Example

```csv
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,EFFECTIVE_DATE,DOB,LAST_NAME,FIRST_NAME,SEX,RACE,ETHNICITY,COUNTY_CODE,CHILD_WELFARE_INVOLVEMENT,TRAILS_IDENTIFIER
CLI,A,,2024-01-15,1990-05-15,Smith,Jane,F,05,2,031,0,
CLI,A,CLT-000002,01/15/2024,1990-05-15,Garcia,,X,99,5,999,1,
```

**Errors:**
- Row 1: `CLI-001` - BHASO_CLIENT_IDENTIFIER is required
- Row 2: `FLD-002` - EFFECTIVE_DATE must be in YYYY-MM-DD format
- Row 2: `CLI-003` - FIRST_NAME is required
- Row 2: `FLD-003` - SEX value 'X' not in allowed values
- Row 2: `FLD-003` - RACE value '99' not in allowed values
- Row 2: `FLD-003` - COUNTY_CODE '999' not valid
- Row 2: `XFD-001` - TRAILS_IDENTIFIER required when CHILD_WELFARE_INVOLVEMENT = 1

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `CLI-001` | BHASO_CLIENT_IDENTIFIER is required | Provide unique client identifier |
| `CLI-002` | DOB must be valid date in YYYY-MM-DD format | Check date format |
| `CLI-003` | FIRST_NAME is required | Provide first name |
| `CLI-004` | LAST_NAME is required | Provide last name |
| `CLI-005` | SEX is required | Provide sex at birth code |
| `CLI-006` | RACE is required | Provide at least one race code |
| `CLI-007` | ETHNICITY is required | Provide ethnicity code |
| `CLI-008` | COUNTY_CODE is required | Provide valid Colorado county code |
| `CLI-009` | Duplicate BHASO_CLIENT_IDENTIFIER | Use Update (U) for existing clients |
| `CLI-010` | DOB indicates age > 120 years | Verify date of birth |
| `CLI-011` | DOB is in the future | DOB cannot be after today |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Child Welfare | `CHILD_WELFARE_INVOLVEMENT`, `TRAILS_IDENTIFIER` | If `CHILD_WELFARE_INVOLVEMENT` = `1`, then `TRAILS_IDENTIFIER` required |
| Medicaid | `MEDICAID_ELIGIBLE`, `MEDICAID_ID` | If `MEDICAID_ELIGIBLE` = `1`, then `MEDICAID_ID` required |
| Age Validation | `DOB` | Calculated age must be 0-120 years |
| Address Consistency | `ADDRESS_LINE_1`, `CITY`, `STATE`, `ZIP_CODE` | If any address field provided, CITY, STATE, ZIP required |

## 6. Update Semantics

### Primary Key

`BHASO_CLIENT_IDENTIFIER` + `EFFECTIVE_DATE`

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates new client record; fails if client+effective_date exists |
| Update | `U` | Creates new effective-dated version of existing client |
| Replace | `R` | Overwrites most recent client record (error corrections) |

### Idempotency

- Most recent record determined by `EFFECTIVE_DATE`
- System uses SCD Type 2 - all historical records retained
- Query for "current" client returns record with latest `EFFECTIVE_DATE`
