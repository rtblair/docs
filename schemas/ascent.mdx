## Overview

The ASCENT schema captures data for youth residential treatment programs. It consists of three record types:
- **ASA** - Ascent Admission: Initial assessment at program entry
- **ASQ** - Ascent Quarterly: Quarterly progress assessments
- **ASD** - Ascent Discharge: Final assessment at program completion

<Note>
This schema is **only required** for youth enrolled in residential treatment programs. Must be linked to an existing ADMISSION record.
</Note>

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `ASA-001` | ASCENT_ADMISSION_DATE is required | Provide admission date |
| `ASA-002` | PLACEMENT_TYPE is required | Provide valid placement code |
| `ASA-003` | ASA record must be submitted before ASQ or ASD | Submit admission first |
| `ASQ-001` | QUARTER_NUMBER must be sequential | Check quarter numbering |
| `ASQ-002` | ASA record not found | Submit ASA record first |
| `ASD-001` | DISCHARGE_DESTINATION is required | Provide destination code |
| `ASD-002` | ASA record not found | Submit ASA record first |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Record Sequence | ASA → ASQ → ASD | Must follow admission order |
| Quarter Sequence | `QUARTER_NUMBER` | Must be sequential (1, 2, 3...) |
| Date Order | All dates | Quarterly dates must be after admission |
| Discharge | ASD | Cannot have ASD without ASA |

## 6. Update Semantics

### Primary Keys

| Record Type | Primary Key |
|-------------|-------------|
| ASA | `BHASO_CASE_IDENTIFIER` (one per admission) |
| ASQ | `BHASO_CASE_IDENTIFIER` + `QUARTER_NUMBER` |
| ASD | `BHASO_CASE_IDENTIFIER` (one per admission) |

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates new record |
| Update | `U` | Updates existing record |
| Replace | `R` | Overwrites record |
