## Overview

The SC PREGNANCY RISK schema captures risk assessment and monitoring data for pregnant and postpartum women in the Special Connections (SC) program. This record documents risk factors, prenatal care, and maternal health indicators throughout the program.

<Note>
This schema is **only required** for Special Connections program participants. Submit PRS records at intake and periodically throughout enrollment to track risk changes.
</Note>

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `PRS-001` | ASSESSMENT_DATE is required | Provide assessment date |
| `PRS-002` | PREGNANCY_STATUS is required | Provide status (1, 2, or 3) |
| `PRS-003` | Invalid ASSESSMENT_TYPE | Use I, P, or D |
| `PRS-004` | Admission not found | Submit ADMISSION record first |
| `PRS-005` | GRAVIDA must be ≥ PARA | Check pregnancy history |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Pregnant Fields | `PREGNANCY_STATUS`, `ESTIMATED_DUE_DATE`, `GESTATIONAL_AGE` | Required if pregnant |
| MAT Details | `MAT_CURRENT`, `MAT_MEDICATION`, `MAT_DOSE` | If on MAT, medication and dose recommended |
| Risk Factors | `HIGH_RISK_PREGNANCY`, `HIGH_RISK_FACTORS` | If high risk, factors required |
| Prenatal Care | `PRENATAL_CARE`, `PRENATAL_VISITS` | If care, visits should be tracked |

## 6. Update Semantics

### Primary Key

`BHASO_CASE_IDENTIFIER` + `ASSESSMENT_DATE` + `ASSESSMENT_TYPE`

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates new assessment record |
| Update | `U` | Updates existing assessment |
| Replace | `R` | Overwrites assessment record |

### Assessment Frequency

- **Intake (I)**: One per admission
- **Periodic (P)**: Monthly or as clinically indicated
- **Discharge (D)**: One at program completion
