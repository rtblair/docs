## Overview

The SUBSTANCE USE DISORDER schema captures detailed information about substances used by clients. There are two record types:
- **SIA** - Substance Use at Admission: Substance use patterns at intake
- **SID** - Substance Use at Discharge: Substance use status at end of treatment

Multiple substances can be recorded per admission/discharge, with one designated as primary.

## 1. Schema Reference

### Required Fields

| BHA Column Name | Field Name | Format | Valid Values | Constraints | Example |
|-----------------|------------|--------|--------------|-------------|---------|
| `RECORD_TYPE` | Record Type | `x(3)` | `SIA` or `SID` | Must match context | `SIA` |
| `TRANSACTION_TYPE` | Transaction Type | `x(1)` | `A`, `U`, `R` | See [Transaction Types](/getting-started/transaction-types) | `A` |
| `BHASO_CLIENT_IDENTIFIER` | Client Identifier | `x(20)` | Alphanumeric | Must match admission | `CLT-000001` |
| `BHASO_CASE_IDENTIFIER` | Case Identifier | `x(20)` | Alphanumeric | Must exist in ADMISSION | `CSE-000001` |
| `EFFECTIVE_DATE` | Effective Date | `date` | YYYY-MM-DD | Cannot be future | `2024-01-15` |
| `SUBSTANCE_CODE` | Substance Code | `x(3)` | See codes below | Valid substance | `001` |
| `PRIMARY_SUBSTANCE` | Primary Flag | `x(1)` | `0` or `1` | One per admission/discharge | `1` |
| `FREQUENCY_USE` | Frequency of Use | `x(2)` | See codes below | Usage pattern | `03` |
| `ROUTE_ADMIN` | Route of Administration | `x(2)` | See codes below | How substance used | `01` |
| `AGE_FIRST_USE` | Age at First Use | `9(2)` | 1-99 | Years old | `16` |

| Code | Description |
|------|-------------|
| `01` | No use in past month |

| `03` | 1-2 times per week |

| Code | Description |
|------|-------------|

### Conditional Fields

| BHA Column Name | Field Name | Format | Condition | Constraints | Example |
|-----------------|------------|--------|-----------|-------------|---------|
| `DAYS_USED_30` | Days Used (Last 30) | `9(2)` | Required for SIA | 0-30 | `15` |
| `DAYS_ABSTINENT_30` | Days Abstinent (Last 30) | `9(2)` | Required for SID | 0-30 | `28` |

### Optional Fields

| BHA Column Name | Field Name | Format | Default | Constraints | Example |
|-----------------|------------|--------|---------|-------------|---------|
| `SUBSTANCE_NAME` | Substance Name | `x(100)` | NULL | Specific name | `Oxycodone` |
| `DETAILED_USE_PATTERN` | Use Pattern | `x(200)` | NULL | Description | `Weekend use only` |
| `PRIOR_TREATMENT_SUBSTANCE` | Prior SUD Treatment | `x(1)` | `0` | `0` or `1` | `1` |

| `MAT_MEDICATION` | MAT Medication | `x(3)` | NULL | Code value | `001` |
| `WITHDRAWAL_SYMPTOMS` | Withdrawal Symptoms | `x(1)` | NULL | `0` or `1` | `1` |
| `NOTES` | Clinical Notes | `x(500)` | NULL | Free text | `History of overdose` |

### Invalid Example

```csv
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,BHASO_CASE_IDENTIFIER,EFFECTIVE_DATE,SUBSTANCE_CODE,PRIMARY_SUBSTANCE,FREQUENCY_USE,ROUTE_ADMIN,AGE_FIRST_USE,DAYS_USED_30
SIA,A,CLT-000001,CSE-999999,2024-01-15,002,1,04,01,16,20
SIA,A,CLT-000001,CSE-000001,2024-01-15,999,1,04,01,16,20
SIA,A,CLT-000001,CSE-000001,2024-01-15,002,1,04,01,16,
SIA,A,CLT-000001,CSE-000001,2024-01-15,002,1,04,01,16,45
```

**Errors:**
- Row 1: `REF-002` - Admission not found: CSE-999999
- Row 2: `SIA-003` - Invalid SUBSTANCE_CODE: 999
- Row 3: `XFD-001` - DAYS_USED_30 required for SIA record type
- Row 4: `SIA-004` - DAYS_USED_30 must be 0-30

## 3. Validation Rules

### Error Codes

| Code | Message | Resolution |
|------|---------|------------|
| `SIA-001` | SUBSTANCE_CODE is required | Provide valid substance code |
| `SIA-002` | FREQUENCY_USE is required | Provide usage frequency |
| `SIA-003` | Invalid SUBSTANCE_CODE | Use code from valid list |
| `SIA-004` | DAYS_USED_30 must be 0-30 | Check value range |
| `SIA-005` | Multiple primary substances | Only one can be PRIMARY_SUBSTANCE=1 |
| `SIA-006` | No primary substance | At least one must be PRIMARY_SUBSTANCE=1 |
| `SID-001` | DAYS_ABSTINENT_30 required for SID | Provide days abstinent |
| `SID-002` | DAYS_ABSTINENT_30 must be 0-30 | Check value range |

### Cross-Field Validation

| Rule | Fields | Constraint |
|------|--------|------------|
| Primary Substance | All substances for case | Exactly one must have `PRIMARY_SUBSTANCE` = `1` |
| Days Used (SIA) | `RECORD_TYPE`, `DAYS_USED_30` | `DAYS_USED_30` required when `RECORD_TYPE` = `SIA` |
| Days Abstinent (SID) | `RECORD_TYPE`, `DAYS_ABSTINENT_30` | `DAYS_ABSTINENT_30` required when `RECORD_TYPE` = `SID` |
| IV Drug Use | `IV_DRUG_USE`, `SHARED_NEEDLES` | If `IV_DRUG_USE` = `1`, `SHARED_NEEDLES` should be answered |
| MAT | `MAT_CURRENT`, `MAT_MEDICATION` | If `MAT_CURRENT` = `1`, `MAT_MEDICATION` recommended |

## 6. Update Semantics

### Primary Key

`BHASO_CASE_IDENTIFIER` + `SUBSTANCE_CODE` + `RECORD_TYPE` + `EFFECTIVE_DATE`

### Transaction Types

| Type | Code | Behavior |
|------|------|----------|
| Add | `A` | Creates new substance record |
| Update | `U` | Updates existing substance (same code, new effective date) |
| Replace | `R` | Overwrites most recent substance record |

### Multiple Substances

- Multiple substances per admission/discharge are allowed
- Each substance identified by unique `SUBSTANCE_CODE`
- One substance must be marked as `PRIMARY_SUBSTANCE` = `1`
- Use code `001` (None) for clients without substance use issues
