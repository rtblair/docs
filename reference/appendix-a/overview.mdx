## Overview

Appendix A contains all valid code values used throughout the CoBHRM Unified Data Model. When submitting data, you must use the exact codes specified in these tables.

<Warning>
Using invalid codes will result in validation errors. Always refer to the current version of Appendix A for valid values.
</Warning>

## Code Table Format

Each code table includes:

| Column | Description |
|--------|-------------|
| **Code** | The exact value to use in your data file |
| **Description** | Human-readable meaning of the code |
| **Effective Date** | When this code became valid (if applicable) |
| **Retired Date** | When this code was retired (if applicable) |
| **Notes** | Additional context or usage guidance |

## Using Code Tables

### Single-Value Fields

For fields that accept a single code value, use the code exactly as shown:

```csv
SEX,COUNTY_CODE,REFERRAL_SOURCE
F,031,01
```

### Multi-Value Fields

Some fields accept multiple values separated by semicolon and space (`; `):

```csv
RACE,DISABILITY_CODES
05; 06; 07,01; 03
```

<Tip>
The delimiter for multi-value fields is semicolon followed by a single space: `; `
</Tip>

## Requesting New Codes

If you need a code value that doesn't exist in Appendix A:

1. Contact the BHA Data Team
2. Provide justification for the new code
3. Temporary workaround: Use code `99` (Other) with explanation in notes field
