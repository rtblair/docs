## Template Usage Guide

### Step 1: Download the Template

Download the template for the schema you need to submit.

### Step 2: Review Field Requirements

Each template includes a reference row with:
- `R` = Required field
- `C` = Conditionally required field
- `O` = Optional field

### Step 3: Prepare Your Data

1. Delete the sample row (keep only header row)
2. Map your data to the template columns
3. Ensure all required fields have values
4. Check conditional requirements

### Step 4: Validate Format

Before submitting:
- [ ] File is saved as UTF-8 CSV
- [ ] All required fields have values
- [ ] Date fields are in YYYY-MM-DD format
- [ ] Code values match Appendix A

## Template Format

All templates follow this format:

```csv
# Row 1: Column headers
RECORD_TYPE,TRANSACTION_TYPE,BHASO_CLIENT_IDENTIFIER,...

# Row 2 (optional): Field requirements (R=Required, C=Conditional, O=Optional)
R,R,R,...

# Row 3: Sample data
CLI,A,CLT-000001,...
```

<Warning>
Delete rows 2 and 3 before submitting your data. Only the header row and your data rows should be in the final file.
</Warning>

## Troubleshooting

### Common Template Issues

| Issue | Solution |
|-------|----------|
| Column mismatch error | Use the official template - don't rename or reorder columns |
| Encoding error | Save as UTF-8, not UTF-16 or ANSI |
| Extra columns | Remove any columns not in the template |
| Missing header row | First row must be the column headers |
