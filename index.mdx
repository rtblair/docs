## Welcome

The Colorado Behavioral Health Reporting Model (CoBHRM) defines the standardized format for submitting behavioral health data to the Colorado Behavioral Health Administration (BHA).

This documentation provides complete specifications for CSV-based data imports, including field definitions, validation rules, and example files.

## Schema Quick Reference

The UDM consists of 9 schemas organized into three categories:

### Core Schemas

| Schema | Record Type | Description |
|--------|-------------|-------------|
| [CLIENT](/schemas/client) | `CLI` | Client demographics and identifiers |
| [ADMISSION](/schemas/admission) | `ADM` | Treatment episode admission records |
| [DISCHARGE](/schemas/discharge) | `DIS` | Treatment episode discharge records |

### Clinical Data

| Schema | Record Types | Description |
|--------|--------------|-------------|
| [DIAGNOSIS](/schemas/diagnosis) | `DXA`, `DXD` | Diagnosis at admission and discharge |
| [SUBSTANCE USE DISORDER](/schemas/substance-use-disorder) | `SIA`, `SID` | Substance use information |

### Specialized Programs

| Schema | Record Types | Description |
|--------|--------------|-------------|
| [ACT](/schemas/act) | `ACT` | Assertive Community Treatment |
| [ASCENT](/schemas/ascent) | `ASA`, `ASQ`, `ASD` | Youth residential treatment |
| [SC BABY](/schemas/sc-baby) | `BAB` | Special Connections - Baby outcomes |
| [SC PREGNANCY RISK](/schemas/sc-pregnancy-risk) | `PRS` | Special Connections - Pregnancy risk assessment |
